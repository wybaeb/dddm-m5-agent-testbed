# Бот-воркспейс студента — практика Модуля 5

Персональный **ReAct-агент**, к которому студент обращается через Telegram/Max-бота.
Каждый студент авторизуется логином и паролем, получает **свой изолированный
рабочий каталог** и общается с агентом обычными сообщениями: «построй график
выручки по месяцам», «найди топ-5 городов», «разберись, почему просела выручка».

Это **упрощённая копия того же харнесса**, на котором работают взрослые
кодинг-агенты (Claude Code, Cursor, Cline): цикл «Мысль → Действие → Наблюдение»
поверх function calling. Мы оставили только суть, чтобы была видна концепция.

```
студент ──Telegram/Max──▶ bot.py ──▶ Agent (ReAct-цикл) ──▶ инструменты:
  логин/пароль            poll +        agent_core.py         python  (sandbox.py)
  свой воркспейс          авторизация                         list_files
  видит лог агента                                            sql (read-only)
                                                              ▲
                                          LLM-роутер курса (OpenAI-совместимый)
```

## Из чего состоит

| Файл | Назначение |
|------|-----------|
| `bot.py` | Long-polling бот: авторизация, сессии, маршрутизация в агента, отправка графиков |
| `agent_core.py` | Ядро ReAct-агента (Мысль → Действие → Наблюдение) на function calling |
| `sandbox.py` | Инструменты агента + изоляция: `python`, `list_files`, `sql` (read-only) |
| `local_chat.py` | Тот же агент в терминале — для отладки/грейдинга без Telegram |
| `tasks/` | Набор аналитических заданий, которые студент поручает агенту |
| `users.example.json` | Шаблон логинов/паролей (пароли — sha256) |

## Запуск

```bash
cp users.example.json users.json   # завести студентов (пароли уже захешированы)
cp .env.example .env               # вписать ключ LLM-роутера и токен бота
./run.sh
```

- **Ключ LLM-роутера** (`AI_PROVIDER_TOKEN`) выдаёт LMS/КТС — Anthropic не нужен.
- **Токен бота** (`BOT_TOKEN`) получается у [@BotFather](https://t.me/BotFather).
- Без Telegram можно проверить агента сразу: `python3 local_chat.py "построй график выручки по месяцам"`.

Демо-доступ из `users.example.json`: логин `demo`, пароль `demo`.

## Что демонстрирует (привязка к урокам М5)

- **5.1 ReAct** — цикл «Мысль → Действие → Наблюдение» виден студенту прямо в чате,
  каждый шаг прозрачен и воспроизводим.
- **5.4 Claw-like** — на задании 6 агент сам декомпозирует цель, проверяет гипотезу
  и итерирует (автономное исследование, а не одна команда).
- **5.5 Безопасное встраивание** — авторизация, изоляция воркспейса на студента,
  таймаут исполнения, read-only гардрейл на SQL. В проде песочница = контейнер на
  студента; здесь учебная (подпроцесс + cwd + таймаут).

## Как добавить студентов

`users.json` — словарь `логин → {name, password_sha256}`. Хеш пароля:

```bash
python3 -c "import hashlib;print(hashlib.sha256('ПАРОЛЬ'.encode()).hexdigest())"
```

## Бот в Max вместо Telegram

Транспорт вынесен в класс `Transport` (bot.py). Для мессенджера **Max** меняется
только `base_url` и имена методов Bot API — логика поллинга, авторизации и агента
остаётся та же. Это и есть смысл: харнесс не зависит от канала.
