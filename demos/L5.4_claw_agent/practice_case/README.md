# Кейс 5.4 — «Окупаемость каналов и удержание когорт»

Практический кейс, на котором видно главное отличие **Claw-like (харнесс) агента**
от обычного ReAct: получив бизнес-задачу, агент **сам пишет код в своей песочнице,
перерабатывает два источника данных (один — грязный) и собирает файл-отчёт**.

## Что в папке

| Файл | Роль |
|------|------|
| `PROMPT.md` | Постановка задачи — то, что вы отдаёте агенту в чат |
| `gen_spend.py` | Генератор второго, «грязного» источника `marketing_spend.csv` |
| `marketing_spend.csv` | Помесячные расходы на маркетинг (готовый, можно перегенерить) |
| `reference_solution.py` | **Эталон для методиста** (планка результата). Студенту/агенту НЕ выдаётся |

Первый источник — общий `../../shared/data/sales_data.csv` (заказы за 2024).

## Как запустить (на харнессе из урока 5.8)

Кейс рассчитан на бот-воркспейс `demos/L5.8_workspace_bot` — там у агента есть
изолированный рабочий каталог и инструмент `python` (он и есть «песочница»).

```bash
# 1) подготовить грязный источник
python3 gen_spend.py

# 2) положить оба файла в рабочий каталог агента (workspace студента),
#    например для локальной отладки без Telegram:
cd ../../L5.8_workspace_bot
cp ../shared/data/sales_data.csv ../L5.4_claw_agent/practice_case/marketing_spend.csv ./workspaces/<логин>/  2>/dev/null || true
export AI_PROVIDER_TOKEN=...
python3 local_chat.py     # вставьте текст задачи из PROMPT.md
```

Через Telegram/Max то же самое: студент пишет агенту текст из `PROMPT.md`, агент
работает в своём воркспейсе и присылает картинку-теплокарту + ссылается на
собранные файлы (`report.html`, `EXECUTIVE_SUMMARY.md`).

Проверить, что кейс решаем, и увидеть планку:

```bash
python3 reference_solution.py
# → retention_heatmap.png · report.html · EXECUTIVE_SUMMARY.md
```

## Ожидаемые артефакты от агента

1. **Журнал сверки** грязного источника (что нормализовал: ~9 написаний каналов,
   1 дубль, 1 пропуск, 3 формата дат).
2. **Экономика каналов** — таблица CAC / LTV / ROMI по каналам.
3. **Матрица удержания когорт** + тепловая карта `*.png` (уходит в чат).
4. **Аномалии расхода** по IQR (ключевая находка — перегрев бюджета одного канала).
5. **Файл-отчёт** `report.html`/`.xlsx` + `EXECUTIVE_SUMMARY.md` с рекомендацией.

## На что смотреть (чему учит)

- Агент **пишет программу, запускает её и итерирует** на реальном выводе — это не
  визуализация ради графика, а программная обработка данных под задачу.
- Ему пришлось **переработать исходные данные** (распарсить грязь, перестроить в
  когорты) — без своей ФС и кода это не сделать.
- Результат — **набор файлов-артефактов**, а не одна реплика в чат.

## Критерии (метод, не кейс)

Оцениваем **способ**, а не конкретные числа: честно ли почищен источник (есть журнал
сверки), посчитаны ли CAC/ROMI кодом из данных, построена ли когортная матрица,
найдены ли аномалии обоснованным методом (IQR/там, где применимо), собран ли
файл-отчёт с рекомендацией. На своих данных студента критерии работают так же.
