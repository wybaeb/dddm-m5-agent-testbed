# Эталонный лог ReAct-агента (для разбора — ДЗ Модуля 5, часть 1)

Вопрос пользователя: **«Какой город принёс больше всего выручки за 2024 год?»**

Ниже — сырой лог работы агента. В задании нужно:
1. промаркировать каждый шаг как **Мысль / Действие / Наблюдение**;
2. найти **ошибочный шаг** (агент допустил одну содержательную ошибку);
3. объяснить, **как агент пришёл к итоговому ответу** — и почему этот ответ
   нельзя принимать на веру.

> ⚠️ В логе намеренно оставлена ровно одна ошибка. Финальный ответ выглядит
> уверенно, но получен неверно. Найдите, где именно агент свернул не туда.

---

```
[1] Чтобы ответить, мне нужно понять структуру данных: какие есть столбцы,
    есть ли выручка и город. Сначала загляну в файл.

[2] python:
    import pandas as pd
    df = pd.read_csv(DATA_PATH)
    print(df.columns.tolist())

[3] ['order_id', 'date', 'customer_id', 'city', 'channel',
     'product_category', 'product', 'quantity', 'price', 'revenue']

[4] Отлично, есть столбцы city, quantity и revenue. Чтобы найти город-лидер,
    сгруппирую данные по городу и просуммирую объём продаж.

[5] python:
    import pandas as pd
    df = pd.read_csv(DATA_PATH)
    top = df.groupby('city')['quantity'].sum().sort_values(ascending=False)
    print(top.head(3))

[6] city
    Москва           41210
    Санкт-Петербург  18940
    Новосибирск       9870
    Name: quantity, dtype: int64

[7] Лидирует Москва с большим отрывом по объёму. Значит, и по выручке
    город-лидер — Москва. Дам ответ.

[8] Финальный ответ: Больше всего выручки за 2024 год принёс город Москва —
    с явным отрывом от Санкт-Петербурга и Новосибирска.
```

## Подсказка для проверяющего (эталон разбора, в задании студенту не показывается)

- **Маркировка:** [1] Мысль, [2] Действие, [3] Наблюдение, [4] Мысль,
  [5] Действие, [6] Наблюдение, [7] Мысль, [8] Финальный ответ.
- **Ошибочный шаг — [5] (и закрепляющая его мысль [4]).** Вопрос был про
  **выручку**, а агент сгруппировал и просуммировал **`quantity`** (количество
  единиц), а не **`revenue`**. Объём в штуках и выручка в деньгах — разные вещи:
  город с большим числом дешёвых заказов может уступать по выручке городу с
  меньшим числом дорогих. Корректный шаг: `df.groupby('city')['revenue'].sum()`.
- **Как агент дошёл до ответа:** на шаге [7] он молча приравнял «лидер по
  количеству» к «лидеру по выручке» и не проверил это на данных. Ответ [8] может
  случайно оказаться верным, но он **не обоснован** — выручку агент так и не
  посчитал. Правильное поведение ReAct: пересчитать по `revenue` и только тогда
  делать вывод.
